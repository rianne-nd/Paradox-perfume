# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

1. **Do NOT** open a public issue
2. Email the details to the repository maintainer
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Supported Versions

Currently, we support the latest version deployed on GitHub Pages.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Security Best Practices

This project follows these security practices:

- No sensitive data stored in the repository
- Dependencies are regularly reviewed
- Code is reviewed before merging
- GitHub Actions workflows use pinned versions

## Disclosure Policy

When we receive a security bug report, we will:

1. Confirm the problem and determine affected versions
2. Audit code to find similar problems
3. Prepare fixes and release them as quickly as possible

Thank you for helping keep Paradox Manila secure!
